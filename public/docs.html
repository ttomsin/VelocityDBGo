VelocityDBGo Documentation

Welcome! VelocityDBGo is a dynamic Backend-as-a-Service (BaaS). It lets you build your frontend web or mobile applications instantly without having to code a backend server, setup a SQL database, or worry about migrations.

Just create a Project, generate an API Key, and start sending JSON data directly from your app.

Authentication (API Keys)

To access the Data API from your client-side application (like React, Vue, or Vanilla JS), you must use your Project's permanent API Key.

You can find this key by logging into the Dashboard and selecting your project.

Include the key in the HTTP headers of every fetch request:

fetch('http://localhost:8080/api/projects/1/data/users', {
  headers: {
    'Content-Type': 'application/json',
    'X-API-Key': 'YOUR-API-KEY-HERE'
  }
})

1. Insert Data (Create)

To store data, simply POST an arbitrary JSON object to your collection endpoint. The collection is created virtually on the fly.

Endpoint

POST /api/projects/{projectId}/data/{collectionName}

Example

const response = await fetch('/api/projects/1/data/restaurants', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json', 'X-API-Key': 'YOUR-API-KEY' },
  body: JSON.stringify({
    name: "Pizza Palace",
    rating: 4.8,
    location: { city: "New York" }
  })
});

2. Query via URL (Read & Filter)

You can retrieve and filter your data using intuitive URL query parameters.

Endpoint

GET /api/projects/{projectId}/data/{collectionName}?key=value

Supported Operators

  • eq: (Equals) - Default if no prefix is given. e.g., ?status=active
  • neq: (Not Equals) - e.g., ?status=neq:closed
  • gt: (Greater Than) - e.g., ?rating=gt:4
  • gte: (Greater or Equal) - e.g., ?rating=gte:4.5
  • lt: (Less Than) - e.g., ?price=lt:100
  • lte: (Less or Equal) - e.g., ?price=lte:100
  • like: (Fuzzy Search/Contains) - Case insensitive. e.g., ?name=like:pizza

Nested Lookups, Sorting & Pagination

Use dot notation for nested JSON fields: ?location.city=New York

Sort: ?sort=rating:desc or ?sort=name:asc

Pagination: ?limit=10&offset=0

3. Query via JSON (Advanced)

If your URL queries become too long, or you prefer a structured approach in your code, you can use the POST /query endpoint.

Endpoint

POST /api/projects/{projectId}/data/{collectionName}/query

Example Payload

{
  "filter": {
    "location.city": "New York",
    "rating": "gte:4.5"
  },
  "sort": "rating:desc",
  "limit": 10
}

4. Update Data (Replace)

Fully replace the JSON payload of an existing document using its ID.

Endpoint

PUT /api/projects/{projectId}/data/{collectionName}/{docId}

Example

const response = await fetch('/api/projects/1/data/restaurants/5', {
  method: 'PUT',
  headers: { 'Content-Type': 'application/json', 'X-API-Key': 'YOUR-API-KEY' },
  body: JSON.stringify({
    name: "Pizza Palace (Updated)",
    rating: 5.0
  })
});

5. Delete Data

Remove a specific document from a collection.

Endpoint

DELETE /api/projects/{projectId}/data/{collectionName}/{docId}

Returns a 200 OK message upon successful deletion.
